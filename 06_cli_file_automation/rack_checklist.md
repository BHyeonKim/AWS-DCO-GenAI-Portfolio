# [교육용 샘플 데이터 사용] SAMPLE Rack Checklist & Log Verification

> [!IMPORTANT]
> **본 문서는 교육용 샘플 데이터를 사용하고 있으며, 실제 AWS 내부 정보, 실제 장비명, 실제 IP 주소, 실제 고객 정보 또는 실제 절차를 포함하고 있지 않습니다.**

이 체크리스트는 가상의 로그 파일 및 모의 티켓 정보를 확인하고 기록하기 위한 교육용 체크리스트입니다. 실제 장비나 프로덕션 환경이 아닌 샘플 데이터 검토용으로만 활용됩니다.

---

## 1. 기본 정보 확인 항목 (Basic Verification)
* [ ] **발생 시간 확인 (Verify Event Time)**
  * 샘플 로그/티켓에 명시된 최초 에러 발생 시점 기록: `[ 예시: 2026-07-07 14:00:00 KST ]`
* [ ] **샘플 장비명 확인 (Verify Sample Hostname)**
  * 점검 대상이 되는 샘플 장비명이 정확히 일치하는지 대조: `[ 예시: SAMPLE_TOR_SW_01 ]`
* [ ] **Ticket 상태 확인 (Verify Ticket Status)**
  * 티켓이 현재 `Open` 상태인지, 다른 작업자에 의해 처리 중인지 대조: `[ 예시: Open ]`

## 2. 세부 분석 및 분석 주의사항 (Analysis Guidelines)
* [ ] **반복 여부 확인 (Verify Recurrence)**
  * 최근 로그 및 과거 이력 검토를 통해 동일 샘플 장비 혹은 동일 포트에서 CRC 에러 및 Link Down이 빈번하게 반복되었는지 확인.
* [ ] **원인 단정 금지 (Do Not Pre-determine the Cause) ⚠️**
  * **중요:** 단순히 로그 상에서 `Link Down` 또는 `CRC Error`가 관찰되었다고 해서 물리적인 케이블 불량이나 트랜시버(Transceiver) 고장으로 바로 속단하지 말 것.
  * 가상 장비 구성 변경 이력, 상위 장비(Spine/Core)의 상태, 혹은 소프트웨어 포트 설정 오류일 가능성을 배정하고 단계별 검토를 수립할 것.

## 3. 보고서 초안 작성 메모 (Report Drafting Notes)
* [ ] **모의 작업 내용 정리**
  * 발견된 타임라인, 장비명, 시나리오 정보를 기반으로 보고서 초안 작성 시 포함할 항목 정리.
  * 예시 내용 작성 시 어떠한 경우에도 실제 정보(실제 IP, 장비 시리얼, 비밀번호 등)를 기재하지 않도록 주의.
